# 08 — PVE 9 基础环境实施清单

> 目标：在已升级到 PVE 9 的前提下，按"LXC 承担基础设施，VM 承担 Docker 宿主机"的路线，逐步完成 homelab 基础环境落地。
>
> 原则：
> - 先稳定、后扩展
> - 先基础设施、后业务应用
> - 先 Compose 主工作流、后 k3s 演进
> - 每一步都要可验证、可回滚、可记录

---

## 数据目录约定

**所有服务数据目录统一以 `/data/` 开头：**

| 服务 | 数据目录 |
|---|---|
| MySQL / MariaDB | `/data/mysql` |
| Redis | `/data/redis` |
| MinIO | `/data/minio` |
| Gitea | `/data/gitea` |
| Prometheus | `/data/prometheus` |
| Grafana | `/data/grafana` |

---

## 0. 当前阶段目标

本阶段完成后，应具备以下能力：

- PVE 9 宿主基础状态稳定
- 存储规划明确
- 网络与 IP 策略明确
- 第一批 LXC 基础设施可用（MySQL、Redis）
- 具备最小备份能力

---

## 1. 宿主机基线检查

### 1.1 系统状态确认
- [ ] 确认 PVE 9 升级完成且可正常重启
- [ ] 确认 Web 管理界面正常访问
- [ ] 确认宿主机时间、时区、NTP 同步正常
- [ ] 确认系统更新已完成
- [ ] 确认宿主机 SSH 可正常登录

### 1.2 资源状态检查
- [ ] 记录 CPU / 内存 / 磁盘当前使用情况
- [ ] 确认根盘剩余空间是否足够
- [ ] 确认是否已有额外数据盘可用于业务数据
- [ ] 记录已有 LXC / VM，避免命名和 IP 冲突

### 1.3 宿主机内核参数（Redis 优化需在宿主机执行）
- [ ] 配置 `vm.overcommit_memory = 1`（见 Redis 手册）

---

## 2. 存储与备份准备

### 2.1 存储规划
- [ ] 区分 PVE 系统盘、业务数据盘、备份盘
- [ ] 明确 LXC 数据挂载点（`mp0`）落点存储池
- [ ] 避免数据库数据堆在系统盘

### 2.2 备份策略
- [ ] 确定 PVE 层 LXC 备份策略（周期、保留数量）
- [ ] 确定数据库逻辑备份脚本（mysqldump）
- [ ] 确定 Redis 持久化文件备份策略
- [ ] 备份位置与系统盘分离

---

## 3. 网络与 IP 规划

### 3.1 IP 分配规划（示例）

| CT ID | 服务 | 建议 IP |
|---|---|---|
| 201 | mysql-01 | 192.168.1.201 |
| 202 | redis-01 | 192.168.1.202 |
| 203 | minio-01 | 192.168.1.203 |
| 204 | gitea-01 | 192.168.1.204 |
| 205 | monitor-01 | 192.168.1.205 |

> 根据实际网段修改，保持固定 IP，便于服务间互联。

### 3.2 DNS 规划
- [ ] 确定内网域名规则（如 `mysql.home`、`redis.home`）
- [ ] 配置本地 hosts 或局域网 DNS

## 4. LXC 基础设施部署顺序

> 详细参数见 [09-lxc-service-resources.md](09-lxc-service-resources.md)

### 4.1 第一批推荐顺序
1. [ ] **redis-01** — 最轻，适合验证 LXC 流程（见 [11-lxc-redis-setup.md](11-lxc-redis-setup.md)）
2. [ ] **mysql-01** — 核心数据依赖，早建早稳（见 [10-lxc-mysql-setup.md](10-lxc-mysql-setup.md)）
3. [ ] **minio-01** — 对象存储
4. [ ] **gitea-01** — 代码版本管理
5. [ ] **monitor-01** — 监控（可后置）

### 4.2 每个 LXC 通用检查项
- [ ] 创建前确认名称、IP、CPU、内存、磁盘规划
- [ ] 创建后确认容器可登录
- [ ] 配置静态 IP
- [ ] 配置时区（`Asia/Shanghai`）、NTP
- [ ] 确认 `/data/<服务名>` 挂载点可写
- [ ] 配置开机自启：`systemctl enable <服务>`
- [ ] 配置 PVE 层备份策略
