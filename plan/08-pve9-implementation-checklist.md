# PVE 9 基础环境实施清单

> 目标：在已升级到 PVE 9 的前提下，按"LXC 承担基础设施，VM 承担 Docker 宿主机"的路线，逐步完成 homelab 基础环境落地。
>
> 原则：
> - 先稳定、后扩展
> - 先基础设施、后业务应用
> - 先 Compose 主工作流、后 k3s 演进
> - 每一步都要可验证、可回滚、可记录

---

## 2. 存储与备份准备

### 2.1 存储规划
- [ ] 区分 PVE 系统盘、业务数据盘、备份盘
- [ ] 明确 LXC 数据目录统一落点（见：[数据目录规范](#22-数据目录规范)）
- [ ] 明确 Docker VM 的数据盘规划
- [ ] 明确镜像、卷、日志的存放位置
- [ ] 避免数据库数据和 Docker 镜像全部堆在系统盘

### 2.2 数据目录规范

所有基础设施 LXC 的业务数据目录统一使用 `/data/<service>` 前缀：

| 服务 | 容器内数据目录 | 说明 |
|------|--------------|------|
| MySQL | `/data/mysql` | datadir，替换默认 `/var/lib/mysql` |
| Redis | `/data/redis` | 持久化目录，替换默认 `/var/lib/redis` |
| MinIO | `/data/minio` | 对象存储数据目录 |
| Gitea | `/data/gitea` | 仓库与配置目录 |
| Prometheus | `/data/prometheus` | 时序数据目录 |
| Grafana | `/data/grafana` | 仪表盘与配置 |

> 约定原则：
> - `rootfs` 只放系统和软件包，**不放业务数据**
> - 业务数据通过 PVE `mp0` 挂载点放在独立磁盘卷上
> - 容器内统一通过 `/data/<service>` 路径访问
> - 方便迁移、备份和排查

### 2.3 备份策略
- [ ] 确定 PVE 层快照 / 备份策略
- [ ] 确定 LXC 备份周期
- [ ] 确定 VM 备份周期
- [ ] 确定数据库逻辑备份方式（如 dump）
- [ ] 确定 MinIO / 上传目录 / Git 数据备份方式
- [ ] 确定备份保留周期
- [ ] 确定至少一份异地或异盘备份位置

### 2.4 验证要求
- [ ] 至少演练一次 LXC 恢复
- [ ] 至少演练一次数据库备份恢复
- [ ] 明确"可恢复"标准，而不是只有备份文件
